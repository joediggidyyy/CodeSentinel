## How to Contribute

### Reporting Issues

- Use the GitHub issue tracker to report bugs or suggest features
- Check existing issues before creating a new one
- Provide detailed information about the issue or feature request

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes
4. Commit your changes with clear, descriptive messages
5. Push to your branch (`git push origin feature/your-feature`)
6. Open a Pull Request

### Code Style

Code style guidelines will be established during the migration process.

## Migration Process

This repository is being prepared to host the CodeSentinel application. If you're involved in the migration:

1. Coordinate with the repository maintainers
2. Ensure all necessary files and history are transferred
3. Update documentation to reflect the new repository location
4. Test all functionality after migration

## Questions?

If you have questions about contributing or the migration, please open an issue or contact the maintainers.
